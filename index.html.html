مقدمة مطوَّلة — لماذا تتعلم البرمجة والأمن السيبراني؟

البرمجة ليست مجرد كتابة أسطر كود — هي طريقة تفكير. عندما تتعلم البرمجة تتعلّم كيف تفكك المشكلات إلى أجزاء صغيرة، وكيف تبني حلولًا قابلة للتكرار والصيانة. في عالم اليوم، الفهم التقني يمنحك حرية أكبر في العمل والتعليم والابتكار. تخصص الأمن السيبراني يضيف بعدًا أخلاقيًا ومهنيًا: القدرة على حماية الأنظمة من الأخطار، وفهم كيفية تصميم أنظمة تتمتع بمستوى أمان عالي.

الهدف من هذا الملف هو أن يمنحك خريطة طريق واضحة، شرحًا مبسَّطًا، أمثلة تعليمية قابلة للتشغيل محليًا، وروحًا تحفيزية باسم No Body — لأن الاسم يعني القوة الصامتة، والتحرك الذكي خلف الستار.

خريطة التعلم المقترحة (مفصّلة)

المرحلة الأولى — الأساسيات (أسابيع 1–8): تعلم أساسيات لغة برمجة (مثل Python)، المفاهيم العامة: المتغيرات، الحلقات، الشروط، الدوال، وكيفية استخدام الطرفية (Terminal) لإدارة الملفات.

المرحلة الثانية — مشاريع صغيرة (أسابيع 9–20): اصنع مشاريع عملية صغيرة — صفحة ويب تفاعلية، API بسيط، سكربت للتعامل مع ملفات — ودوّن كل خطأ وتعلم من كل تجربة.

المرحلة الثالثة — التركيز على الأمن والممارسة الآمنة (أسابيع 21–32): تعلّم مبادئ الأمن، كتابة كود آمن، تصميم اختبارات داخل بيئة محلية معزولة، والمشاركة في تحديات قانونية داخل مختبرات تعليمية.

أساسيات البرمجة (شرح مُفصّل)

ابدأ بفهم المتغيرات (Variables): هي أماكن تخزين مؤقتة للقيم. بعد ذلك تعلّم الشروط (if / else) للحكم على شروط، والحلقات (for / while) لتكرار العمل، والدوال (functions) لتجميع منطق قابل لإعادة الاستخدام.

# مثال Python: متغيرات وشروط وحلقة
count = 0
names = ['Ali', 'Sara', 'NoBody']
for name in names:
    if 'No' in name:
        print(f'Found: {name}')
    count += 1
print('Total:', count)

هذا المثال يعرّف قائمة ويبحث عن اسم يحتوي على 'No' ويعطي نتيجة. مبادئ بسيطة لكنها أساس لكل شيء أكبر لاحقًا.

البرمجة الكائنية OOP — شرح واضح

الفكرة: تجميع البيانات والوظائف ضمن كائن واحد (Object). هذا يساعد على تنظيم المشاريع الكبيرة.

# مثال Python: كلاس بسيط
class Service:
    def __init__(self, name):
        self.name = name
    def info(self):
        return f'Service: {self.name}'

s = Service('NoBodyScanDemo')
print(s.info())

قواعد البيانات — المفهوم وتطبيق آمن

تعلّم الفرق بين قواعد البيانات العلاقية (مثل SQL) وغير العلاقية (NoSQL). الأهم هو كيفية الاتصال بها بأمان وعدم تمرير مدخلات المستخدم مباشرةً في الاستعلامات.

# Python + sqlite3 — مثال آمن باستخدام معاملات parameterized
import sqlite3
conn = sqlite3.connect(':memory:')
cursor = conn.cursor()
cursor.execute("CREATE TABLE users (id INTEGER PRIMARY KEY, username TEXT)")
username = "nobody"
cursor.execute("INSERT INTO users (username) VALUES (?)", (username,))
cursor.execute("SELECT * FROM users WHERE username = ?", (username,))
print(cursor.fetchall())

الويب — مفاهيم أساسية بدون تفاصيل ضارة

تعلم ما معنى طلب HTTP، رأس Request، جسم Body، حالة Status Code، وكيف تُقدّم صفحة ويب. هذه المعرفة تساعدك على فهم كيف تُبنى التطبيقات وكيف يجب حمايتها.

أمن التطبيقات — مفاهيم الحماية

نقدّم وصفًا نظريًا للثغرات الشائعة وطرق الحماية. هدفنا تعليمي ومَنَع الضرر.

XSS — نظرة عامة وحماية

الـ XSS يحدث عندما تُعرض بيانات مستخدم دون ترميز، مما يسمح بتشغيل نص برمجي في متصفح الضحية. الحماية: ترميز/هروب البيانات عند العرض، فحص المدخلات، وتفعيل سياسات CSP قوية عند النشر.

SQL Injection — نظرة عامة وحماية

يحدث عندما تدخل استعلامات SQL من دون حماية. حماية فعّالة تكون عبر معاملات parameterized، والحد من صلاحيات قواعد البيانات.

CSRF — نظرة عامة وحماية

هجوم يقوم بإيصال طلبات نيابةً عن المستخدم دون علمه. الحماية: رموز CSRF في النماذج، وفحص Origin/Referer، واستخدام طرق تحقق إضافية في العمليات الحساسة.

ممارسات التطوير الآمن — خطوات بسيطة

  • استخدم التحقق من المدخلات دائماً (Validate & Sanitize).
  • لا تعرض معلومات داخلية في رسائل الخطأ (وحدّث سجلاتك محليًا فقط).
  • ادعم التشفير للقنوات عند النشر (HTTPS).
  • حدّ من صلاحيات الحسابات وقواعد البيانات.

مشروعات تطبيقية (تفصيل لكل مشروع)

لكل مشروع أذكر أهدافًا ومخرجات متوقعة:

  1. صفحة هبوط تفاعلية: HTML/CSS + JS، نموذج يحفظ البيانات محليًا. هدف: فهم DOM والتفاعل.
  2. API صغير: اكتب API محليًا يستقبل بيانات ويعيد JSON. هدف: فهم الطلبات والاستجابات.
  3. تطبيق مؤمن: أضف حماية ضد XSS وCSRF، وتأكد من اختبار المدخلات.
